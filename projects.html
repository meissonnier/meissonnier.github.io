Contexte & Objectifs

Conception d'une architecture multi-sites sécurisée pour une entreprise fictive. Mise en place d'un réseau résilient avec segmentation logique et sécurisation des accès distants.

Objectifs Critiques
  • Segmentation VLAN par service.
  • Interconnexion de 3 sites distants via VPN IPsec.
  • Redondance des liens (HSRP, EtherChannel).

Architecture & Plan d'Adressage

Optimisation du plan d'adressage via VLSM.

Plan des VLANs (Siège Social)
  • VLAN 10 (Admin) : 10.0.10.0/24
  • VLAN 20 (Compta) : 10.0.20.0/24
  • VLAN 99 (Management) : 10.0.99.0/24
Sécurité Périphérique

Déploiement d'un Pare-feu ASA. Configuration du NAT (Dynamic PAT) et création d'une zone DMZ pour les serveurs Web accessibles depuis l'extérieur.

Configurations Techniques (IOS)

! Configuration VPN Phase 1 (ISAKMP)
crypto isakmp policy 10
 encr aes 256
 hash sha256
 authentication pre-share
 group 14
crypto isakmp key SECRETKEY address 203.0.113.2

! Configuration Phase 2 (IPsec)
crypto ipsec transform-set IPSEC-SET esp-aes 256 esp-sha256-hmac

interface GigabitEthernet0/1
 description UPLINK_VERS_CORE
 switchport mode trunk
 switchport trunk allowed vlan 10,20,99
 spanning-tree portfast trunk

Automatisation avec Nmap & Bash

Le script effectue un scan discret, parse les résultats XML, et les compare avec une base de données de CVE connues.

#!/bin/bash
TARGET=$1
echo "[+] Démarrage du scan furtif sur $TARGET"
nmap -sS -sV -T3 -oX report.xml $TARGET
python3 parse_and_vuln.py report.xml > final_report.html
echo "[+] Scan terminé. Rapport HTML généré."
